## Confirmed Fixes

### Fix 1 — Bar Chart Height in `app/(app)/reports/page.tsx`

**Problem**: The CSS bar chart bars use `height: X%` inside a `div` with `height: auto`. Since there is no definite height on the containing block, `height: X%` resolves to `height: auto` = 0px. Bars are invisible.

**Root cause comparison**: The dashboard bar chart (which works) uses `style={{ height: "100%" }}` on the bar wrapper. Chrome resolves `100%` against the flex container's definite height (`h-44` = 176px), giving the wrapper a definite height. The bars then correctly render at `X%` of 176px.

**Fix**: In the `weekData.map(...)` loop, add `style={{ height: "100%" }}` to the bar wrapper div:

```jsx
{/* Before */}
<div className="flex w-full items-end">
  <div className="w-full rounded-t bg-blue-500/70" style={{ height: `${day.pct}%` }} />
</div>

{/* After */}
<div className="flex w-full items-end" style={{ height: "100%" }}>
  <div className="w-full rounded-t bg-blue-500/70" style={{ height: `${day.pct}%` }} />
</div>
```

**Scope**: 1-line change in `reports/page.tsx`. No structural changes.

### Fix 2 — Customer Card Text Truncation in `app/(app)/customers/page.tsx`

**Problem**: At 375px viewport, the contact email text (e.g., "mariam@example.com" at ~130px) exceeds the available flex-1 column width (~115px), causing it to overflow its parent.

**Fix**: Add `truncate` class to the name, contact, and lastOrder text divs. The parent already has `min-w-0 flex-1` which is required for `truncate` to work inside flexbox.

```jsx
{/* Before */}
<div className="text-sm font-semibold text-white">{customer.name}</div>
<div className="text-xs text-gray-500">{customer.contact}</div>
<div className="text-xs text-gray-400">Last order: {customer.lastOrder}</div>

{/* After */}
<div className="truncate text-sm font-semibold text-white">{customer.name}</div>
<div className="truncate text-xs text-gray-500">{customer.contact}</div>
<div className="truncate text-xs text-gray-400">Last order: {customer.lastOrder}</div>
```

**Scope**: 3-line change in `customers/page.tsx`. No structural changes.
